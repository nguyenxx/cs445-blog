React for Dummies: Fundamentals

Last Updated: October 29, 2020

In preparation for the React App project, in addition to watching the lecture videos, I decided to read some articles and summarize the fundamental React.js concepts, I've learned so far.

Components, Props, States

According to the article "React for Dummies" by Kelly Harrop, components are the building blocks of your React app. You form bigger components by putting together smaller components. According to the article All the fundamental React.js concepts, jammed into this one article by Samer Buna, a component name starts with a capital letter since lowercase case names are reserved for HTML elements. The ReactDOM will ignore the functioin, if that is the case.

Just like HTML elements, every component in React has a list of attributes, often referred to as props - the data that came from the parent to the child. On the other hand, if the data that is owned and managed by the child that is called a state. In our lecture, we created a Swatch component for our Pal app in the swatch.js file. We added to our Swatch function a parameter props, which is an object of all the data that gets sent. In our case, we recieved a color object (props.color).

JSX

According to the document React Fundamentals by React Native, JSX is a syntax that lets you write elements inside JavaScript. If you want to write a JavaScript expression inside a JSX section, you will need to use that expression inside the curly braces to make it work. You can also use the self-closing form of the tag, if there are no children. Kelly Harrop describes JSX as something that "looks like HTML in your JS file". Kelly says the most obvious difference is the use of className insread of class when defining your elements. The use of the JSX syntax has also been demonstrated in our lecture video.

Events in React

According to a React document, there are two significant differences from the way we handle events inside React elements vs. with the DOM API:

  • First, all React element attributes are names using camelCase rather than lowercase.
  • Secondly, JavaScript function reference is passed as the event handler, rather than a string

HTML vs. React

                    <button onclick="activateLasers()">
                    Activate Lasers
                    </button>
                
                    <button onclick={activateLasers}>
                    Activate Lasers
                    </button>

Another difference that the document notes is that you cannot return false to prevent default behavior in React. You must call preventDefault explicitly.
